Overview

This is a subproject of jcel and contains its core. This means that all the classification algorithms are implemented in this subproject. Besides, classes and properties are coded as integer numbers. Axioms are also simple structures based on integer numbers. The main class is the Processor. Once it is configured, it is possible to run the classification algorithm.
The steps to follow for an instance of Processor should be:

  • configure<>
  • process all queues while there is a change
  • post-process the data creating a graph with direct subsumers, direct subsumee, and equivalent vertices

Package dependencies

If we think each package as a node, and each dependency between two packages as a directed edge, the structure of package dependency should be a directed acyclic graph (DAG). The packages should only make public those classes that are or could be used by another package preserving the DAG structure.

Overview

This program is a reasoner based on the CEL algorithms. The CEL system is known for its scalability of reasoning in the lightweight description logic EL++. This logic has been proved suitable for several ontology applications, most notably from the life science domain. The description logic EL++ has been adopted as the logical basis of the OWL 2 EL profile of the new Web Ontology Language, integrating the reasoner to the OWL user community.

Main steps of the classification algorithm.

  • translate OWL API symbols into integer numbers
  • run the configuration, which:
    • normalizes the ontology creating auxiliary entities
    • creates an extended ontonlogy based on the normalized ontology
    • adds the concepts
    • creates the role hierarchy
    • prepares all the queues to run the algorithm
  • while there is any change, process all the queues.
  • remove auxiliary entities
  • build a graph containing direct subsumers, direct subsumees and equivalents
  • create OWL API data structures based on the integer graph

Package dependencies

If we think each package as a node, and each dependency between two packages as a directed edge, the structure of package dependency should be a directed acyclic graph (DAG). The packages should only make public those classes that are or could be used by another package preserving the DAG structure.
